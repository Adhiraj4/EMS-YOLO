# Critical Perspective - Batch Size 16 Sweep (T=5)
* **Optimization Baseline Replication:** As this sweep uses a batch size of 16 (identical to the [Gen1 Baseline](file:///Users/eugenix/Downloads/Books/Business/Documente/University/Master/Q4/Fundamental%20ML/EMS-YOLO/g1-resnet/runs/test/exp1_baseline/gen1_baseline_results.txt)), it serves as an excellent verification of training stability. It reached a peak validation mAP@.5 of **0.623** at Epoch 10, identical to the baseline run's ceiling of **0.623** (which peaked at Epoch 17).
* **Early Stabilization:** The model quickly established its ceiling, plateauing starting from Epoch 5 (mAP@.5 = 0.597) and entering a narrow band between **0.606** and **0.623** throughout the rest of the run.
* **Batch Size Suite Analysis (Initial LR = 0.01):**
  - **BS=8**: [Batch Size 8 Sweep](file:///Users/eugenix/Downloads/Books/Business/Documente/University/Master/Q4/Fundamental%20ML/EMS-YOLO/g1-resnet/runs/test/exp3_bs_8/bs_8_results.txt) peaked at **0.626 mAP@.5** (Memory: 6.92 GB).
  - **BS=16**: Peaked at **0.623 mAP@.5** (Memory: 13.2 GB).
  - **BS=32**: [Batch Size 32 Sweep](file:///Users/eugenix/Downloads/Books/Business/Documente/University/Master/Q4/Fundamental%20ML/EMS-YOLO/g1-resnet/runs/test/exp3_bs_32/bs_32_results.txt) peaked at **0.622 mAP@.5** (Memory: 25.8 GB).
  - **BS=64**: [Batch Size 64 Sweep](file:///Users/eugenix/Downloads/Books/Business/Documente/University/Master/Q4/Fundamental%20ML/EMS-YOLO/g1-resnet/runs/test/exp3_bs_64/bs_64_results.txt) peaked at **0.620 mAP@.5** (Memory: 50.7 GB).
  - *Finding:* Increasing batch size scales up memory usage exponentially but actually yields a slight, steady decrease in validation mAP@.5 (from 0.626 down to 0.620). This indicates that smaller batch sizes (specifically BS=8) optimize better for Spiking Neural Networks on sparse event frames, likely because smaller batches introduce beneficial stochastic gradient noise.
